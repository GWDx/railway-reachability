# Railway Reachability Engine

面向中国铁路时刻表的**时间可达性分析引擎**。

## 三个核心查询

| 查询                 | 问题                 | 示例                                           |
| -------------------- | -------------------- | ---------------------------------------------- |
| **Earliest Arrival** | 最早什么时候能到？   | "20:00 从合肥南出发，最早几点到上海虹桥？"     |
| **Reachability**     | DDL 前能到吗？       | "13:00 在合肥南，16:00 前能到上海虹桥吗？"     |
| **Latest Departure** | 最晚什么时候必须走？ | "23:59 前要到上海虹桥，最晚几点从合肥南出发？" |

每个查询同时给出**直达**和**换乘**两种方案，并标注每段行程的出发/到达时间、换乘等待时间。

## 项目结构

```
├── engine.py              # 核心引擎：EAT 算法、可达性查询、最晚出发

│   └── train_stops.jsonl  # 每趟列车的完整停站 + 时刻
└── README.md
```

## 快速开始

### 安装

```bash
pip install py12306
```

### 查询

```bash
# 单 OD 对查询
python query.py

# 多站组合查询（如 合肥南/合肥 → 上海虹桥/上海，自动选最优）
python query_multi.py
```

## 算法

### Earliest Arrival Time（EAT）

优先队列驱动的 **Label-Setting** 算法，核心思路：

1. 从起点站出发，按到达时间顺序探索
2. 在每个站，乘坐所有赶得上的列车（出发时间 ≥ 到达时间 + 换乘缓冲）
3. 沿列车到达每个后续站，若更早到达则更新并加入队列
4. DDL 剪枝：超过 DDL 的状态不再探索

时间复杂度：$O((S + T \cdot L) \log S)$，其中 $S$ = 车站数，$T$ = 列车数，$L$ = 平均停站数。

### Latest Departure

对出发时间做二分搜索，每次调用 EAT 验证是否能在 DDL 前到达。复杂度 $O(\log M \cdot \text{EAT})$，$M$ 为 DDL 分钟数。

### 直达查询

线性扫描所有列车，找同时经停起点和终点的车次，记录最早/最晚时间。$O(T \cdot L)$。
